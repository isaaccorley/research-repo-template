# Why Research Decks?

A slide system that is:

- **Version controlled** — slides live in Git, reviews via PRs
- **On-brand** — purple/blue dark theme, custom fonts, and gradients baked in
- **Developer-friendly** — write Markdown, drop in React components
- **Self-contained** — no Google Slides, no Figma exports, no context switching

> Write once, present anywhere. Export to PDF with `?print=true`.

<SpeakerNotes>
Research Decks was born from the need to stop copy-pasting code screenshots into Google Slides.
Now code is live-rendered with syntax highlighting and diagrams stay in sync with the codebase.
</SpeakerNotes>

# How It Works

Each deck is a single **MDX file** — Markdown + JSX:

```markdown
# My Slide Title

- Bullet points in **Markdown**
- Inline `code` works too

---

<SectionSlide>

## A Big Statement

</SectionSlide>
```

Separate slides with `---`. That's it.

<SpeakerNotes>
MDX files live in the pages/ directory. Next.js handles routing automatically.
The Deck component splits content on horizontal rules to create individual slides.
</SpeakerNotes>

# Live Code Blocks

Syntax highlighting with the custom color palette:

```python
import torch
import torch.nn as nn

class ResidualBlock(nn.Module):
    def __init__(self, channels: int):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, 3, padding=1)
        self.bn1 = nn.BatchNorm2d(channels)
        self.conv2 = nn.Conv2d(channels, channels, 3, padding=1)
        self.bn2 = nn.BatchNorm2d(channels)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        residual = x
        out = torch.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        return torch.relu(out + residual)
```

<SpeakerNotes>
Code blocks use shiki with a custom theme. Python, SQL, TypeScript, and more are supported out of the box.
No screenshots needed — the code is always up to date.
</SpeakerNotes>

## Built-in Components

Every deck has access to:

- **TitleSlide** — cover page with logo and gradient title
- **SectionSlide** — big centered divider text
- **Columns** — 2 or 3 column grid layouts
- **CodeBlock** — syntax-highlighted code fences
- **Embed** — full-viewport iframes for live demos
- **Diagram** — SVG diagrams with theme-aware colors
- **SpeakerNotes** — hidden notes, toggle with **N**

| Key | Action |
|-----|--------|
| `→` `↓` `Space` | Next slide |
| `←` `↑` | Previous slide |
| `N` | Toggle speaker notes |
| `F` | Toggle fullscreen |

### URL Params

- `?print=true` — render all slides for PDF export
- `?notes=true` — show speaker notes
- `#3` — jump to slide 3

# Getting Started

```bash
# Install dependencies
bun install

# Start the dev server
bun run dev

# Edit pages/decks/*.mdx — your deck auto-reloads
```

Create a **branch per deck**. Open a PR for review.

Export a static site with `bun run export` — deploy anywhere.

<SpeakerNotes>
Each presentation lives on its own branch. The main branch stays as the template.
Static export means you can host on S3, GitHub Pages, Vercel, or just open the HTML files.
</SpeakerNotes>
